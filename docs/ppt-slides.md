## 슬라이드 3: 프로젝트 배경 - 현재 채용 업무의 문제점

### 레이아웃
- 상단: 제목
- 중앙: 프로세스 흐름도 (다이어그램)
- 하단: 핵심 Pain Points 3개 (카드형)

### 비주얼 - 채용 프로세스 흐름도

```
┌─────────────┐    수동 확인    ┌─────────────┐    수동 입력    ┌─────────────┐    메일만 발송    ┌──────────┐
│  마이다스인   │ ──────────→ │    구글       │ ──────────→ │   VNTG      │ ─────────────→ │  후보자   │
│  채용플랫폼   │             │  스프레드시트  │             │   채용웹     │                │          │
└─────────────┘             └─────────────┘             └─────────────┘                └──────────┘
   지원서 접수                  전형별 트래킹                결과 안내 발송
                                   │
                              수식으로 확인
                                   ↓
                            ┌──────────────┐
                            │  채용 현황 파악  │
                            └──────────────┘
```

### 핵심 Pain Points (카드 3장)

| Pain Point | 설명 |
|:---|:---|
| 3개 툴 분산 운영 | 채용플랫폼, 스프레드시트, 채용웹을 번갈아가며 사용 |
| 모든 작업이 수동 입력 | 전형 결과, 날짜, 후보자 정보를 모두 수기로 입력 |
| 실시간 현황 파악 불가 | 스프레드시트 수식에 의존, 입력이 늦으면 현황 파악 불가 |

### 상세 설명 (하단 또는 다음 슬라이드 연결용)

- 마이다스인 채용플랫폼
  - 지원서 접수 시 자동으로 스크리닝 전형에 등록되지만,
  - 서류/역량검사/1차면접/2차면접 전형은 채용담당자가 전형을 직접 생성하고 대상자를 일일이 수동 추가해야 함
  - 공채(정기 채용)에 특화된 시스템이라 수시채용 방식과 맞지 않음

- 구글 스프레드시트
  - 신규 지원서 접수 확인 → 스프레드시트에 수동 등록
  - 각 전형별 결과(합격/불합격) 발생 시 결과 + 날짜 수동 입력
  - 리드타임 계산은 수식에 의존
  - 진행 상황 파악도 수식 기반

- VNTG 채용웹
  - 후보자에게 전형별 결과를 안내할 때만 사용
  - 채용웹에 해당 후보자를 별도 등록해야 메일 발송 가능
  - 메일 보내기 외에 채용 관련 기능은 없음

### 발표 노트
> 현재 채용 업무의 프로세스를 말씀드리겠습니다. 마이다스인 채용플랫폼에서 지원서가 접수되면, 채용담당자가 이를 확인하여 구글 스프레드시트에 수동으로 등록합니다. 전형별 결과가 나올 때마다 스프레드시트에 합격/불합격과 날짜를 수기로 입력하고, 리드타임은 수식으로 계산합니다. 후보자에게 결과를 안내할 때는 또 다시 VNTG 채용웹에 후보자를 등록한 뒤 메일을 발송해야 합니다. 특히 마이다스인 플랫폼은 공채에 특화되어 있어서 수시채용 방식인 저희와 맞지 않고, 스크리닝 이후 전형에서는 전형 생성부터 대상자 추가까지 모두 수동입니다.

## 슬라이드 4: 프로젝트 배경 - 기존 채용 툴의 한계 (상세)

### 레이아웃
- 3열 비교표 (카드형 or 테이블)
- 각 도구별 아이콘 상단 배치

### 비교표

| 구분 | 마이다스인 채용플랫폼 | 구글 스프레드시트 | VNTG 채용웹 |
|:---|:---|:---|:---|
| **역할** | 지원서 접수 | 전형별 트래킹 | 결과 안내 메일 발송 |
| **문제점** | 수시채용 미지원 / 전형 수동 생성 / 대상자 수동 추가 | 수동 입력 / 실시간성 없음 / 휴먼에러 | 메일 발송만 가능 / 채용 관리 기능 부재 |
| **후보자 경험** | 개인정보, 학력, 경력을 날짜까지 모두 수동 입력 | - | - |
| **자동화 수준** | 스크리닝 접수만 자동 | 없음 (수식만) | 없음 |
| **데이터 연동** | 타 시스템과 연동 없음 | 별도 수동 관리 | 별도 수동 등록 |

### 핵심 메시지 (하단 강조 문구)

```
채용담당자가 3개 도구를 넘나들며 동일한 정보를 반복 입력
→ 리소스 낭비 + 휴먼에러 + 실시간 현황 파악 불가
```

### 발표 노트
> 이 표를 보시면 각 도구의 한계가 명확합니다. 채용플랫폼은 지원서 접수는 되지만 수시채용에 맞지 않고, 스프레드시트는 수동 입력에 의존하며, 채용웹은 메일만 보낼 수 있습니다. 결국 채용담당자가 3개 도구를 오가면서 같은 정보를 반복 입력하고 있는 상황입니다. 특히 지원자 입장에서도 개인정보부터 학력, 경력 날짜까지 모두 수동으로 입력해야 하는 번거로움이 있습니다.

## 슬라이드 5: 프로젝트 목표

### 레이아웃
- 좌우 분할: AS-IS (회색/어두운톤) vs TO-BE (밝은/강조톤)
- 중앙에 화살표 (→ Phase 1 통합)

### AS-IS (왼쪽)

```
AS-IS: 3개 도구 분산 운영

┌─────────────────┐
│  채용플랫폼       │  지원서 접수
├─────────────────┤
│  스프레드시트      │  전형별 트래킹
├─────────────────┤
│  채용웹           │  메일 발송
└─────────────────┘

→ 수동 입력, 분산 관리, 실시간성 없음
```

### TO-BE (오른쪽)

```
TO-BE: VNTG ATS 일원화

┌─────────────────────────────┐
│          VNTG ATS            │
│                              │
│  ✓ 후보자 관리               │
│  ✓ 파이프라인 트래킹          │
│  ✓ 이메일 / 일정 자동화       │
│  ✓ AI 일정 조율              │
│  ✓ 실시간 대시보드            │
│  ✓ Activity Timeline         │
└─────────────────────────────┘

→ 자동화, 통합 관리, 실시간 현황 파악
```

### 핵심 메시지

- 스프레드시트 대체 → 전형별 파이프라인 트래킹을 시스템 내에서 자동화
- 채용웹 대체 → 이메일 발송 + 자동 일정 조율 기능 통합
- 궁극적 목표 → 마이다스인 채용플랫폼까지 포함한 완전 일원화 (Phase 2~3)

### 발표 노트
> 이 프로젝트의 목표는 명확합니다. 3개로 분산된 채용 도구를 VNTG ATS라는 하나의 플랫폼으로 통합하는 것입니다. Phase 1에서는 먼저 구글 스프레드시트와 VNTG 채용웹을 대체합니다. 스프레드시트가 담당하던 전형별 트래킹을 시스템이 자동으로 처리하고, 채용웹의 메일 발송 기능에 더해 AI 기반 자동 일정 조율까지 커버합니다.

## 슬라이드 10: 핵심 기능 3 - AI 자동 일정 조율

### 레이아웃
- 상단: 제목 + AS-IS/TO-BE 한줄 비교
- 중앙: 7단계 플로우 다이어그램 (가로 흐름)
- 하단: 후보자 일정 선택 화면 스크린샷 (모바일)

### AS-IS → TO-BE (한줄 비교)

```
AS-IS: 채용담당자가 면접관 캘린더를 하나하나 확인, 메일로 일정 조율 (평균 2~3일)
TO-BE: AI가 캘린더 자동 분석 → 최적 시간대 제안 → 후보자 링크 선택 (수분 내 완료)
```

### 7단계 플로우 다이어그램

```
① 채용담당자        ② AI 엔진          ③ 교집합 추출       ④ 최적화
   일정 조율    →     Google Calendar →   면접관 N명의   →    충돌 감지 +
   요청 클릭          자동 분석            가용 시간대         슬롯 스코어링
                                          교집합 추출

                          ↓

⑦ 웹훅 자동확인    ⑥ Calendar 등록     ⑤ 후보자 선택
   면접관 수락/     ←  Google Calendar  ←   토큰 링크로
   거절 실시간         자동 등록 +          로그인 없이
   감지                면접관 초대          일정 선택
```

### 세부 기능 (핵심 차별화 포인트)

| 기능 | 설명 |
|:---|:---|
| 스마트 스케줄링 | 면접관 N명의 Google Calendar를 분석하여 교집합 시간대 자동 추출 |
| 한국 공휴일/점심시간 제외 | 공휴일, 점심시간(11:30~12:30) 등 자동 필터링 |
| 부분 충돌 허용 | 전체 면접관이 아닌 최소 인원만 가능해도 옵션 제안 |
| 후보자 모바일 선택 | 토큰 기반 링크로 로그인 없이 일정 선택 (모바일 최적화) |
| Google Calendar 자동 등록 | 확정 즉시 면접관 캘린더에 일정 추가 + 초대 메일 |
| 웹훅 기반 실시간 감지 | Google Calendar 웹훅으로 면접관 수락/거절 실시간 확인 |
| 데일리 리마인드 | 미응답 면접관에게 매일 자동 리마인드 메일 발송 |

### 발표 노트
> 이번 프로젝트의 가장 핵심적인 기능인 AI 자동 일정 조율입니다. 기존에는 채용담당자가 면접관 캘린더를 하나하나 확인하고 메일을 주고받으며 일정을 잡았는데, 평균 2~3일이 걸렸습니다. VNTG ATS에서는 채용담당자가 '일정 조율' 버튼만 클릭하면, AI가 면접관들의 Google Calendar를 자동 분석하여 교집합 시간대를 추출합니다. 한국 공휴일과 점심시간은 자동으로 제외하고, 충돌이 있더라도 최소 인원이 가능하면 옵션으로 제안합니다. 후보자에게는 토큰 기반 링크가 전송되어, 로그인 없이 모바일에서 바로 일정을 선택할 수 있습니다. 일정이 확정되면 Google Calendar에 자동 등록되고, 면접관에게 초대 메일이 발송됩니다. 웹훅을 통해 면접관의 수락/거절도 실시간으로 감지하며, 미응답 시 매일 리마인드 메일이 자동 발송됩니다.

TO-BE: 모든 활동을 시간순 통합 표시 (SNS 피드 스타일) + 인라인 액션
```

### 이벤트 타입 범례

```
📋 전형 이동        후보자가 어떤 전형에서 어떤 전형으로 이동했는지
📅 일정 생성/확정   AI가 일정을 생성하고 후보자가 선택하여 확정
✉️ 이메일 발송      채용담당자가 후보자에게 발송한 이메일 내용
💬 코멘트           팀원 간 내부 소통 (코멘트)
⭐ 평가             면접관이 작성한 평가 요약
🔄 상태 변경        후보자 상태가 변경된 기록
👤 실행 주체 표시   누가 어떤 작업을 수행했는지 프로필로 확인
```

### 주요 특징

| 기능 | 설명 |
|:---|:---|
| 통합 히스토리 | 전형 이동, 일정, 이메일, 코멘트, 평가 등 모든 활동을 시간순으로 표시 |
| 실행 주체 표시 | 각 이벤트를 누가 수행했는지 아바타와 이름으로 표시 |
| 인라인 액션 | 일정 카드에서 바로 재조율/취소/삭제 등 액션 수행 가능 |
| 코멘트 소통 | 타임라인 내에서 팀원과 후보자에 대한 내부 소통 가능 |

### 발표 노트
> 다섯 번째, 후보자 타임라인입니다. 기존에는 스프레드시트에 날짜만 기록했기 때문에 '이 후보자가 언제, 누구에 의해, 어떤 이유로 전형이 바뀌었는지' 전체 맥락을 파악하기 어려웠습니다. VNTG ATS의 Activity Timeline은 SNS 피드처럼 모든 활동을 시간순으로 보여줍니다. 전형 이동, 일정 생성, 이메일 발송, 팀원 코멘트 등이 하나의 타임라인에 통합되고, 각 이벤트를 누가 실행했는지도 프로필로 표시됩니다. 일정 카드에서는 바로 재조율이나 취소 같은 액션도 수행할 수 있습니다.

## 슬라이드 14: 데모 시나리오

### 레이아웃
- 7단계 Step-by-Step (좌에서 우, 또는 지그재그)
- 각 단계마다 작은 스크린샷 썸네일

### 시나리오: "신규 후보자 접수부터 면접 일정 확정까지"

```
Step 1                    Step 2                    Step 3

          ↓
Step 4                    Step 5                    Step 6
┌──────────────┐         ┌──────────────┐         ┌──────────────┐

### 각 단계 설명

| Step | 행위자 | 동작 | 화면 |
|:---:|:---|:---|:---|
| 1 | 채용담당자 | 채용 공고 생성 + 드래그앤드롭으로 프로세스 설계 | Jobs > Create |
| 2 | 채용담당자 | 후보자 이름, 이메일, 포지션 지정하여 등록 | Candidates > Add |
| 3 | 채용담당자 | '일정 조율' 버튼 클릭 → AI가 면접관 캘린더 분석 → 옵션 생성 | 후보자 상세 > 일정 조율 폼 |
| 4 | 시스템 | 후보자에게 일정 선택 링크가 포함된 이메일 자동 발송 | 이메일 발송 |
| 5 | 후보자 | 모바일에서 토큰 링크 접속 → 원하는 일정 선택 | 후보자 인터랙션 페이지 |
| 6 | 시스템 | Google Calendar에 면접 일정 자동 등록 + 면접관 초대 메일 | Calendar 등록 |
| 7 | 채용담당자 | Activity Timeline에서 전체 진행 흐름 한눈에 확인 | 후보자 상세 > Timeline |

### 발표 노트
> 실제 사용 시나리오를 보여드리겠습니다. 먼저 채용담당자가 채용 공고를 만들고 프로세스를 설계합니다. 후보자를 등록한 뒤 '일정 조율' 버튼을 클릭하면 AI가 면접관들의 캘린더를 자동 분석하여 가능한 시간대 옵션을 생성합니다. 후보자에게는 일정 선택 링크가 이메일로 자동 발송되고, 후보자는 모바일에서 로그인 없이 원하는 일정을 선택합니다. 일정이 확정되면 Google Calendar에 자동 등록되고 면접관에게 초대 메일이 나갑니다. 채용담당자는 Activity Timeline에서 이 모든 과정을 한눈에 확인할 수 있습니다. 기존에 수동으로 2~3일 걸리던 일정 조율이 수분 내에 완료됩니다.

## 슬라이드 15: 기대 효과 및 Phase 2 로드맵

### 레이아웃
- 상단: AS-IS vs TO-BE 비교표 (기대 효과)
- 하단: Phase 2/3 로드맵 카드

### 기대 효과 비교표

| 항목 | AS-IS | TO-BE (VNTG ATS) | 개선 효과 |
|:---|:---|:---|:---|
| 일정 조율 | 수동 캘린더 대조 + 메일 왕복 (평균 2~3일) | AI 자동 조율 (수분 내 완료) | 업무 시간 90% 단축 |
| 전형 트래킹 | 스프레드시트 수동 입력 | 시스템 자동 기록 + 실시간 대시보드 | 휴먼에러 제거 |
| 결과 안내 | 채용웹 별도 등록 후 발송 | 후보자 상세에서 바로 발송 | 이중 작업 제거 |
| 채용 현황 파악 | 스프레드시트 수식 확인 | Overview 대시보드 실시간 확인 | 실시간성 확보 |
| 히스토리 관리 | 셀에 날짜만 기록 | Activity Timeline 전체 맥락 보존 | 맥락 보존 |
| 사용 도구 수 | 3개 (플랫폼 + 시트 + 채용웹) | 1개 (VNTG ATS) | 도구 일원화 |

### Phase 2/3 로드맵

```
┌───────────────────────────────────┐    ┌───────────────────────────────────┐
│  Phase 2: 협업 강화                │    │  Phase 3: ATS 완성                 │
│                                    │    │                                    │
│  • 이메일 양방향 동기화             │    │  • AI 이력서 파싱                   │
│    (수신 메일도 자동 연동)          │    │    (PDF 업로드만으로 지원 완료)      │
│  • 온라인 면접 평가표               │    │  • 마이다스인 채용플랫폼 완전 대체   │
│    (디지털 스코어카드)              │    │  • 채용 데이터 자산화               │
│  • 팀원 멘션(@) 및 협업 기능       │    │    (분석/인사이트)                  │
│                                    │    │                                    │
└───────────────────────────────────┘    └───────────────────────────────────┘
```

### 발표 노트
> 기대 효과입니다. 가장 큰 변화는 일정 조율인데, 수동으로 2~3일 걸리던 작업이 수분 내로 단축됩니다. 전형 트래킹은 시스템이 자동 기록하므로 휴먼에러가 사라지고, 결과 안내도 후보자 상세에서 바로 발송하므로 이중 작업이 제거됩니다. 그리고 무엇보다 3개 도구가 1개로 일원화되는 것이 핵심입니다. Phase 2에서는 이메일 양방향 동기화, 면접 평가표, 팀원 멘션 등 협업 기능을 강화하고, Phase 3에서는 AI 이력서 파싱을 도입하여 마이다스인 채용플랫폼까지 완전히 대체할 계획입니다.
